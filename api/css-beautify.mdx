---
title: "css_beautify() — JavaScript API"
description: "Programmatic API for beautifying CSS code with js-beautify."
---

The `css_beautify` function reformats and re-indents CSS source code. It is exported as `beautify.css` and `beautify.css_beautify` from the `js-beautify` package.

## Import

<CodeGroup>

```js CommonJS
const beautify = require('js-beautify');

// Both are equivalent:
const result = beautify.css(code, options);
const result2 = beautify.css_beautify(code, options); // legacy alias
```

```js CommonJS (named)
const { css: css_beautify } = require('js-beautify/js');

const result = css_beautify(code, options);
```

```js ESM
import beautify from 'js-beautify';

const result = beautify.css(code, options);
```

</CodeGroup>

## Function signature

```ts
function css_beautify(code: string, options?: CSSBeautifyOptions): string
```

## Parameters

<ParamField path="code" type="string" required>
  The CSS source code string to beautify.
</ParamField>

<ParamField path="options" type="object">
  Configuration object. All fields are optional. Option names use underscores
  (e.g., `indent_size`), corresponding to the CLI flags with dashes replaced by
  underscores (e.g., `--indent-size`).
</ParamField>

## Return value

Returns a `string` containing the beautified CSS code.

## Options

| Option | Type | Default | Description |
|---|---|---|---|
| `indent_size` | number | `4` | Number of spaces per indentation level. |
| `indent_char` | string | `" "` | Character used for indentation. |
| `indent_with_tabs` | boolean | `false` | Indent with tabs instead of spaces. Overrides `indent_size` and `indent_char`. |
| `eol` | string | `"\n"` | End-of-line character(s). Defaults to the first newline found in the file. |
| `end_with_newline` | boolean | `false` | Ensure the output ends with a newline. |
| `brace_style` | string | `"collapse"` | Brace placement style. One of `collapse` or `expand`. |
| `selector_separator_newline` | boolean | `true` | Add a newline between multiple selectors in a rule. |
| `newline_between_rules` | boolean | `true` | Add a blank line between CSS rules. |
| `indent_empty_lines` | boolean | `false` | Keep indentation on otherwise empty lines. |

## Example

```js
const beautify = require('js-beautify');

const ugly = 'body{margin:0;padding:0;font-family:sans-serif;} h1,h2{color:red;font-weight:bold;}';

const result = beautify.css(ugly, {
  indent_size: 2,
  newline_between_rules: true,
  selector_separator_newline: true,
});

console.log(result);
/*
body {
  margin: 0;
  padding: 0;
  font-family: sans-serif;
}

h1,
h2 {
  color: red;
  font-weight: bold;
}
*/
```

## Legacy aliases

| Name | Notes |
|---|---|
| `require('js-beautify').css` | Preferred short form. |
| `require('js-beautify').css_beautify` | Legacy alias. |

## Configuration sources (Node.js only)

When running in Node.js, options are resolved in the following priority order (highest first):

1. Options passed directly to the function call.
2. `jsbeautify_`-prefixed environment variables.
3. A `JSON` config file specified via the `--config` CLI flag.
4. A `.jsbeautifyrc` file found by walking up the directory tree from the current working directory.

<Note>
Language-specific overrides in `.jsbeautifyrc` are supported. Place CSS options under a `"css"` key to apply them only to CSS files.
</Note>

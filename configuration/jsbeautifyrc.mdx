---
title: ".jsbeautifyrc"
description: "Store JS Beautify options in a .jsbeautifyrc JSON file so you don't have to repeat CLI flags on every run."
---

The `.jsbeautifyrc` file is a JSON configuration file that JS Beautify reads automatically. Place it anywhere in the directory tree above your working directory and it will be found without any extra flags.

<Note>
  `.jsbeautifyrc` is supported by the JavaScript (Node.js) implementation only. The Python package does not read this file.
</Note>

## Configuration priority

When multiple configuration sources are present, earlier sources in the list override later ones:

1. CLI arguments
2. `jsbeautify_`-prefixed environment variables
3. File passed via `--config`
4. `.jsbeautifyrc` (searched from `$PWD` upward through the filesystem)

## File format

The file must contain a valid JSON object. Keys match the underscore-style option names from the [Options reference](/configuration/options).

```json .jsbeautifyrc
{
    "indent_size": 2,
    "indent_char": " ",
    "indent_level": 0,
    "end-with-newline": true,
    "indent_with_tabs": false,
    "preserve_newlines": true,
    "max_preserve_newlines": 10,
    "jslint_happy": false,
    "space_after_anon_function": false,
    "brace_style": "collapse,preserve-inline",
    "keep_array_indentation": false,
    "keep_function_indentation": false,
    "space_before_conditional": true,
    "break_chained_methods": false,
    "eval_code": false,
    "unescape_strings": false,
    "wrap_line_length": 0,
    "css": {
        "selector_separator_newline": false
    }
}
```

## File discovery

JS Beautify walks up the filesystem from your current working directory (`$PWD`) until it finds a `.jsbeautifyrc` file or reaches the root. You can place the file at the project root, a monorepo root, or your home directory to apply settings broadly.

```
your-project/
├── .jsbeautifyrc      ← picked up for all files in this tree
├── src/
│   └── index.js
└── styles/
    └── main.css
```

## Explicit config path

Pass `--config` to point to a specific file instead of relying on discovery:

```bash
js-beautify --config path/to/my-config.json foo.js
```

This is useful in CI pipelines or when you maintain multiple config files for different projects.

## Environment variables

Prefix any option name with `jsbeautify_` to set it via an environment variable:

```bash
export jsbeautify_indent_size=2
export jsbeautify_end_with_newline=true
js-beautify foo.js
```

Environment variables override `.jsbeautifyrc` settings but are overridden by CLI flags.

## Language-specific overrides

You can scope options to a specific language using `js`, `css`, or `html` keys. See [Language-specific overrides](/configuration/language-overrides) for the full inheritance model.

```json .jsbeautifyrc
{
    "indent_size": 4,
    "css": {
        "indent_size": 2
    },
    "html": {
        "indent_size": 2
    }
}
```
